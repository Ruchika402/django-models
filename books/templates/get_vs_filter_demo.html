📖 get() vs filter() - Deep Dive

Understanding the key differences and when to use each

📋 filter() Returns QuerySet

books = Book.objects.filter(genre='FIC')
✅ Returns: QuerySet (even if empty or single item)
📊 Type: {{ results.filter_example.type }}
🔢 Count: {{ results.filter_example.count }} items
🔄 Can iterate: Yes
📝 Example items:
{% for book in results.filter_example.items %}   - {{ book.title }}
{% endfor %}
💡 Best for:
  • Getting multiple records
  • Checking if any records exist
  • Chaining with other filters
  • When you're not sure if data exists

🎯 get() Returns Model Instance

book = Book.objects.get(id=1)
✅ Returns: Single model instance
📊 Type: {{ results.get_example.type }}
📝 Value: {{ results.get_example.value }}
🔄 Can iterate: No
⚠️ Raises: DoesNotExist or MultipleObjectsReturned
💡 Best for:
  • Getting a single unique record
  • When you know exactly one record exists
  • Accessing by primary key (id)
  • In detail views with get_object_or_404

Key Differences

Feature filter() get()
Return Type QuerySet Model instance
No results found Empty QuerySet (no error) Raises DoesNotExist ❌
Multiple results found Returns all matching (no error) Raises MultipleObjectsReturned ❌
Can be chained Yes ✅ No ❌
Lazy evaluation Yes ✅ No (executes immediately) ❌
Use .count() Yes ✅ N/A
Use .exists() Yes ✅ N/A

Useful QuerySet Methods

# Get first and last items
first_book = Book.objects.first()
last_book = Book.objects.last()
First book: {{ results.first_last.first|default:"None" }}

# Check existence (more efficient than count())
has_fiction = Book.objects.filter(genre='FIC').exists()
Has fiction books? {{ results.exists_count.has_fiction }}
Number of fiction books: {{ results.exists_count.fiction_count }}
💡 Note: {{ results.exists_count.note }}
# Safe way to use get() in views
from django.shortcuts import get_object_or_404
book = get_object_or_404(Book, id=some_id)
Using get_object_or_404: Automatically returns 404 if not found

⚠️ Common Pitfalls & Solutions

# ❌ WRONG - This will raise error if multiple books exist
book = Book.objects.get(title="Dune") # May raise MultipleObjectsReturned
# ✅ RIGHT - Use filter() when unsure about uniqueness
books = Book.objects.filter(title="Dune") # Safe, returns QuerySet
if books.exists():
first_book = books.first()
# ✅ RIGHT - Use get() with unique fields only
book = Book.objects.get(id=1) # id is always unique
